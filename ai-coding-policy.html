Core Philosophy

We accept and encourage AI-assisted code â€” using tools like GitHub Copilot, Cursor, Claude, ChatGPT, and others â€” but we strictly reject contributions where the engineer acts merely as a copy-paste proxy for AI output.

The difference is simple: AI-assisted means the engineer understands, verifies, and takes ownership of every line. AI-proxied means the engineer blindly submits what the model generated. The former is a productivity multiplier; the latter is a liability.

Every engineer is the sole responsible author of their submitted code, comments, and design decisions â€” regardless of which tools helped produce them.

Law 1 â€” Proof of Verification

AI tools frequently write code that looks correct but fails at execution. "Vibe checks" and green squiggles are not enough.

Req
Every PR with functional changes must include pasted local test output (e.g. cargo test, pytest, npm test). Screenshots or CI links are also acceptable.
Req

Req

Fail

Law 2 â€” No Hallucinations, No Redundancy

Req

Fail

Law 3 â€” The "Explain It" Standard

If you wrote it, you can explain it. No exceptions.

Req

Fail

Law 5 â€” Security & Sensitivity Awareness

AI models have no concept of your security context. The engineer must be the security gate.

Fully Generated

06

Best Practices for AI-Assisted Development

1
Understand before you commit. Read every line the AI generates. If you can't explain it to a teammate, don't submit it.
2
Use AI for acceleration, not replacement. Let AI handle boilerplate, initial drafts, and pattern matching. Keep architecture, edge cases, and security logic in your own hands.
3
Test aggressively. AI-generated code often has "happy path bias." Write edge case tests, error path tests, and boundary tests yourself.
4
Keep PRs focused. Don't dump an entire AI session into one PR. Break changes into logical, reviewable units â€” exactly as you would with hand-written code.
5
Review AI output with fresh eyes. Take a break between generation and review. AI output looks convincing at first glance â€” distance helps you catch errors.
6
Invest in prompt engineering. The quality of AI output correlates with the quality of your prompt. Provide context, constraints, existing patterns, and specific requirements.
7
Maintain your skills. AI tools are powerful but impermanent. Keep your core engineering skills sharp â€” debugging, system design, reading code, and reasoning about complexity.
